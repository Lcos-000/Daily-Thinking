# Daily Thinking

## 简介

本目录用于记录日常技术思考、问题分析和解决方案的探索过程。每个文件记录一个主题的深入思考,包含问题背景、思考过程、追问与纠正,以及最终的方法论沉淀。

## 文件命名规范

- 文件以日期命名,格式为 `YYYY-M-D.md`
- 每个文件对应一个独立的技术主题


## 笔记结构

每篇笔记通常包含以下部分:

1. **背景/原题回顾** - 问题场景描述
2. **第一反应** - 初步想法或解决方案
3. **追问与思考** - 深入探讨问题的不同角度
4. **纠正** - 对错误理解的纠正
5. **方法论沉淀** - 总结可复用的思考框架或解决方案

## 使用方法

- 按日期查阅特定主题的思考记录
- 通过文件名快速定位感兴趣的内容
- 参考方法论沉淀部分获取总结性的技术观点
